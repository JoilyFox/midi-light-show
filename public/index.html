Mappings

Tip: Learn captures the next knob/button you move. “Fade” actions: ms per unit sets the speed indexing — 10 ⇒ value 100 = 1 s (on/off feel), 1000 ⇒ value 100 = 100 s (slow fade).

How it flows

Ableton / Cubase ──MIDI──▶ IAC Driver Bus 1 ──▶ this bridge ──UDP──▶ WiZ lamp (notes / CC) (map · fade · rate-limit)

MIDI from your DAW — or the Ableton MCP tools send_cc/send_note — hits a mapping, which runs an action on a fixture. Slow devices (WiZ) are rate-limited + de-duplicated automatically.

Timing & modifiers

SettingMeaning
ms per unitSpeed indexing for fades: time = value × ms-per-unit. 10 → value 100 = 1 s (on/off feel). 1000 → value 100 = 100 s (slow fade). Same action, different scale.
Time fromWhere the fade duration comes from: this control’s value, a fixed value, or another CC (a second knob sets the time).
Target fromThe on/target brightness: a fixed %, or the note velocity (loud = bright, soft = dim).
TriggerOn press (value ≥ 64 / note-on) · On release (value < 64 / note-off) · On any change (every value — for knobs/continuous).
CurveFade shape: linear · ease-in · ease-out.
Channel / NumberMIDI channel 1–16 or “Any”. Number = CC number or note number.

Ableton MCP commands

Ask Claude to run these — they emit MIDI out AbletonMCP’s output (routed to IAC), driving the mappings above live.

send_cc(cc, value, channel=1)
one CC message — e.g. CC 20 = 64 → color send_cc_ramp(cc, start, end, duration_ms=1000, steps=20, channel=1)
timed CC sweep — e.g. CC 21 0→127 over 2000 ms → a fade send_note(note, velocity=100, channel=1, duration_ms=200)
note on+off — velocity encodes brightness — e.g. note 62 vel 110

Plus stock tools for building song-synced clips: create_midi_track, create_clip, add_notes_to_clip, set_tempo, fire_clip.
